# 决策总览

这份文件只记录已经拍板的关键决策，避免开发过程中反复摇摆。

## 1. 产品定位

这是一个 **课程作业导向的自然语言量化回测 demo**，不是完整生产级交易系统。

优先级排序：

1. 大模型元素明确
2. 工程骨架清晰
3. A 股逻辑合理
4. 演示效果直观
5. 后续可扩展

## 2. 必须覆盖的两类策略

### 模式 A：单股时序策略
示例：
- 5 日均线上穿 20 日均线买入
- 跌破 10 日均线或亏损 8% 卖出

### 模式 B：股票池横截面选股策略
示例：
- 在股票池中每月选过去 20 日涨幅最高的 10 只等权持有

## 3. 统一内核原则

两类策略都统一为：

> 在某个交易日给出目标持仓权重

即：

`target_weights = {symbol: weight}`

这样 Qlib 策略适配层不需要区分“单股策略内核”和“选股策略内核”。优先继承 `WeightStrategyBase`，由 Qlib 负责从目标仓位生成订单、执行回测和输出评测结果。

## 4. 数据默认策略

- 默认主数据接入：Qlib
- 默认数据目录：`~/.qlib/qlib_data/cn_data`
- 默认 region：`cn`
- 默认股票池：Qlib 内置市场名（如 `csi300`）或用户自定义列表
- 暂不依赖 Tushare 特定权限

## 5. LLM 原则

- 只通过 API 调用
- 只做自然语言 → DSL 的解析
- 不直接生成可执行交易代码
- 不直接生成 Qlib YAML 或 Qlib 表达式
- 必须输出 assumptions / warnings
- 必须支持 provider-agnostic 配置

## 6. A 股规则处理原则

### 第一优先级必须做
- 日频
- Qlib 日频执行语义
- 默认使用 Qlib `deal_price` 配置，V1 推荐 `close` 对齐 Qlib 示例；如要模拟“收盘信号、次日开盘成交”，在内核任务中单独验证 `open`/信号滞后实现
- 手续费 / 印花税
- long-only

### 能力支持时启用，否则降级
- 停牌不可交易
- 涨跌停导致无法成交
- 原始价格 / 复权价格分离
- 更细颗粒度 lot size

## 7. 前端原则

- 单页 Streamlit
- 两个 tab：单股 / 选股
- 先重 Qlib 评测结果、解释和数据状态，不做花哨交互

## 8. 质量保证原则

不依赖人工通读大量生成代码，而依赖：

- 明确契约
- golden cases
- 小而有力的测试
- 示例策略
- capability summary

## 9. 第一版明确不做

- 分钟级
- 实盘
- 做空 / 杠杆
- 全市场复杂横截面因子平台
- 公告 / 财报事件策略
- 让 LLM 直接生成策略执行代码
