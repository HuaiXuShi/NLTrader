# Prompt 策略与输出规范

## 核心思想

让模型“少发挥，多映射”。

Parser 的 prompt 不是让模型创作策略，而是让模型在一个受限语法里完成结构化翻译。

## system prompt 必须传达的内容

### 1. 当前支持范围
- A 股
- 日频
- 单股时序 / 股票池横截面
- long-only
- 输出 symbol 使用 Qlib 风格：`SH600036`、`SZ000001`

### 2. 支持的指标
例如：
- SMA
- EMA
- RSI
- MACD
- BOLL_UPPER / LOWER
- RETURN_N
- VOL_MA_RATIO

### 3. 支持的操作符
例如：
- `>`
- `<`
- `cross_above`
- `cross_below`
- `breakout_high`
- `breakdown_low`

### 4. 支持的调仓频率
- daily
- weekly
- monthly

### 5. 不支持范围
- 分钟级
- 公告 / 财报事件
- 自动代码生成
- 做空 / 杠杆

## 输出策略

模型输出必须遵守：

1. 只能输出 JSON
2. 不要输出 markdown code fence
3. 不要输出解释性散文
4. 不要发明未支持字段
5. 对模糊表达要写 assumptions
6. 对缺失前提要写 warnings
7. 不要输出 Qlib YAML、Python 代码或 Qlib 表达式

## 模糊表达处理原则

### 可补默认值的
例如：
- “放量” → `VOL_MA_RATIO(20) > 1.5`
- “RSI 超卖” → `RSI(14) < 30`

### 不应擅自补的
例如：
- “选龙头股”
- “选强势股”
- “低估值”
- “高成长”

这类应该 warning 或拒绝。

## 推荐 human_summary 风格

尽量短而清楚，例如：

- `识别为单股时序策略：5日均线上穿20日均线买入，跌破10日均线或亏损8%卖出。`
- `识别为横截面月度动量策略：在股票池中按20日收益率排序，选前10只等权持有。`

## 推荐 Prompt 模板骨架

```text
你是一个量化策略解析器。
你的任务是把用户的中文策略翻译成受控 JSON DSL。
你只能使用给定的 strategy_kind、字段、指标、操作符和频率。
如果用户表达模糊，请在 assumptions 中写出默认解释。
如果缺少必要信息或请求超出能力范围，请在 warnings 中说明，必要时拒绝。
股票代码必须标准化为 Qlib 风格，例如 SH600036、SZ000001。
不要输出代码，不要输出 markdown，不要输出 JSON 之外的任何文字。
```

## 关于“支持很多策略”的控制方式

不要在 prompt 中告诉模型“尽量支持所有策略”。  
应该告诉模型：

- 在支持清单内尽量完成映射
- 清单外要明确拒绝或警告

这会比“尽量理解所有输入”稳定得多。
