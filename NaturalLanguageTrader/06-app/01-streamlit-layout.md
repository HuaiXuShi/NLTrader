# Streamlit 单页布局

## 设计原则

- 单页即可
- 两个 tab 即可
- 展示优先于炫技
- 不把业务逻辑塞进 UI

## 推荐页面结构

```text
标题区
├── 项目说明
├── 数据源 / 规则说明
└── 当前能力摘要

Tab 1：单股时序策略
Tab 2：股票池横截面策略
```

## Tab 1：单股时序策略

### 输入区
- 股票代码输入框
  - 支持 `SH600036`
  - 兼容 `600036.SH`，提交前标准化
- 中文策略输入框
- 开始日期
- 结束日期
- 运行按钮

### 输出区
1. 解析结果卡片
   - strategy kind
   - human summary
   - assumptions
   - warnings

2. 回测结果卡片
   - total return
   - annualized return
   - max drawdown
   - sharpe
   - num trades

3. 图表区
   - 价格图 + 买卖点
   - 净值曲线

4. 记录区
   - trades table
   - capability summary

## Tab 2：股票池横截面策略

### 输入区
- 股票池来源选择
  - 预置池
  - 手工输入股票列表
- 中文策略输入框
- 开始日期
- 结束日期
- 运行按钮

### 输出区
1. 解析结果卡片
2. 组合结果卡片
3. 组合净值图
4. 调仓记录表
5. 当前/历史持仓表
6. capability summary

## 页面上建议一直显示的说明

### “系统当前能力”
- 数据源：Qlib
- provider_uri
- benchmark：`SH000300`
- 是否启用停牌规则
- 是否启用涨跌停规则
- 是否使用 Qlib 内置股票池 / 用户股票池

### “系统边界”
- 只支持日频
- 只支持 long-only
- 只支持当前 DSL 范围

## 推荐的 demo 默认示例

### 单股示例
预填：
`5日均线上穿20日均线且成交量大于20日均量1.5倍时买入，跌破10日均线或亏损8%卖出`

### 横截面示例
预填：
`每月调仓，在股票池中选择过去20日涨幅最高的10只股票等权持有`
