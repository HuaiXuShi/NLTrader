# 范围与边界

## V1 必须做

### 场景
- A 股
- 日频
- long-only

### 策略模式
- 单股时序策略
- 股票池横截面选股策略

### 自然语言能力
- 使用外部 LLM API
- 输出结构化 DSL
- 输出 assumptions / warnings
- 输出解析类型说明（时序 / 横截面）

### 数据能力
- 默认 Qlib
- 本地 Qlib provider_uri
- Qlib 内置股票池 / 用户自定义股票列表

### 回测能力
- 统一 target-weight 策略接口
- Qlib 回测与 risk_analysis 评测
- Qlib 日频交易规则、费用和涨跌停配置
- 手续费 / 印花税
- 核心绩效指标
- 回测报告对象

### 演示能力
- Streamlit 单页
- 两个 tab
- 解析结果 + 回测结果 + 简要解释

## V1 应该做，但可降级

- 停牌不可交易
- 涨跌停导致无法成交
- 原始价格 / 复权价格分离
- 更完整的调仓记录解释
- 股票池预置模板（如 `csi300`）

## V1 不做

- 分钟级
- 实盘
- 做空、融资融券、期权
- 全市场复杂选股平台
- 基本面 / 财报 / 公告策略
- 自动生成 Python 策略代码
- 前后端分离
- 自建完整行情数据库
- 高精度逐笔 / 分钟级撮合

## 关于“支持很多策略”的解释

V1 的实现原则不是内置很多完整策略，而是支持一小套可组合原语：

### 时序原语
- 均线
- MACD / RSI / BOLL
- N 日收益率
- 成交量相对均量
- 突破 / 下破
- 止损 / 止盈 / 最大持有天数

### 横截面原语
- 过滤
- 打分
- 排序
- top N / bottom N
- 等权构建
- 固定调仓频率

这个边界非常重要：  
**支持很多策略 ≠ 任意中文都能无损翻译。**

## Qlib 版额外边界

- V1 不做模型训练流水线，`qrun + Alpha158 + LightGBM` 只作为后续扩展参考。
- V1 不把自然语言直接翻译成 Qlib YAML 配置，而是先转成本项目 DSL，再由 adapter 转成 Qlib 可执行策略。
- V1 优先解释可复现的回测口径，不追求覆盖 Qlib 全部策略、优化器和工作流模块。
