# 一页纸定义

## 项目名称

A 股自然语言策略回测系统（日频，Qlib 数据与评测版）

## 产品一句话

用户用自然语言描述策略，系统把策略解析为受控 DSL，并在 Qlib A 股日频数据上执行回测与评测，输出结果、解释和关键假设。

## 目标用户

- 课程作业答辩中的老师和同学
- 想快速验证策略想法的学生团队
- 希望体验“自然语言量化回测”的演示用户

## 用户最想看到的东西

- 我说一句中文，系统能理解
- 系统知道这是单股时序还是横截面选股
- 系统能给出回测结果
- 系统能说明 Qlib 数据、交易规则和评测口径
- 系统能解释它是怎么理解我的
- 系统能说明它做了哪些简化

## 两条主链路

### 链路 1：单股时序策略
输入：
- 股票代码
- 中文策略
- 回测区间

输出：
- 解析结果
- assumptions / warnings
- 回测净值
- 买卖点
- 绩效指标
- 交易记录

### 链路 2：股票池横截面策略
输入：
- 股票池（预置 / 自定义）
- 中文策略
- 回测区间

输出：
- 解析结果
- assumptions / warnings
- 组合净值
- 调仓记录
- 持仓变化
- 绩效指标

## 为什么这个项目成立

它兼顾了三件事：

1. 有大模型参与，而且角色清楚
2. 有量化回测，不是纯聊天
3. 工程上可以压缩成一个统一骨架
4. 底层复用 Qlib 数据与评测，减少自研市场细节的风险

## 成功标准

第一版只要做到下面几点，就算成功：

1. 能稳定跑通单股时序策略
2. 能稳定跑通一个股票池选股策略
3. 能把中文策略解析成结构化结果
4. 能明确告诉用户 assumptions 和 warnings
5. 能在单页 UI 中完整演示
6. 能展示 Qlib capability summary 和评测口径

## 最大风险

1. 把“自然语言”做成无限制，导致 parser 不稳定
2. 把“各种策略都支持”理解成无限模板，导致系统爆炸
3. 把数据层做得太重，拖慢开发
4. 把 UI 做得太多，挤占核心引擎时间
5. 没有把自定义 DSL 语义和 Qlib 执行语义对齐，导致回测口径解释不清
