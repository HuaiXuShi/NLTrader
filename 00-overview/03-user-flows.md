# 用户流程与 Demo 路径

## 流程 A：单股时序策略

### 用户输入
- 股票代码：如 `SH600036`（也可输入 `600036.SH`，由 UI 标准化）
- 中文策略：  
  `5日均线上穿20日均线且成交量大于20日均量1.5倍时买入，跌破10日均线或亏损8%卖出`
- 回测区间：如 `2021-01-01 ~ 2024-12-31`

### 系统过程
1. 识别这是单股时序策略
2. 用 LLM 解析为 DSL
3. 提示 assumptions / warnings
4. 通过 Qlib DataProvider 获取该股票日线数据
5. 编译策略并交给 Qlib 适配层运行回测
6. 返回 Qlib 评测结果、交易/持仓记录、指标

### 页面展示重点
- DSL / 解析摘要
- assumptions
- K 线或价格图上的买卖点
- 策略净值
- 绩效指标
- 交易记录表

## 流程 B：股票池横截面策略

### 用户输入
- 股票池：自定义列表 / 预置池
- 中文策略：  
  `每月调仓，在股票池中选择过去20日涨幅最高的10只股票等权持有`
- 回测区间：如 `2021-01-01 ~ 2024-12-31`

### 系统过程
1. 识别这是横截面策略
2. 解析 DSL
3. 显示 assumptions / warnings
4. 通过 Qlib 获取股票池所有标的的历史数据
5. 在每个调仓点计算筛选、排序和组合权重
6. 运行 Qlib 组合回测
7. 返回净值、调仓记录和持仓变化

### 页面展示重点
- 解析摘要
- 股票池来源说明
- capability summary
- 组合净值
- 调仓记录
- 每次持仓列表

## 答辩演示推荐顺序

### 先演示单股
原因：直观、老师更容易看懂。

建议示例：
- 均线交叉 + 放量确认
- 止损退出

### 再演示横截面
原因：展示“可扩展”和“量化味”。

建议示例：
- 月度 top N 动量选股
- 静态池等权持有

## 页面上的诚实说明

建议在 UI 中始终显示一段“本次回测说明”：

- 使用的数据源
- Qlib provider_uri 和 benchmark
- 当前启用的市场规则
- 当前未启用的规则及原因
- 股票池来源（静态 / 用户提供）
- 本次解析中的 assumptions

这会让系统看起来更成熟，也能降低被追问时的风险。
